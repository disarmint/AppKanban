# AppKanban — канбан-доска для отделов компании

Собственное веб-приложение для управления задачами по отделам: продажи, склад, разработка, закупка товара, клиенты/техподдержка, курьеры/водители, касса, гравировка/пломбы. Работает как внутренний планировщик задач для команды из ~30 человек, со своей базой данных (без синхронизации с Google Sheets) и системой ролей.

## Возможности

- **Канбан-доска по отделам** — колонки «Запланировано / В процессе / Завершено», перетаскивание задач (в том числе touch-перетаскивание на мобильных).
- **Роли пользователей** — администратор видит и редактирует всё, сотрудник (member) видит и работает только со своим отделом.
- **Учётные записи** — создаются администратором прямо в приложении (экран «Пользователи»), без внешних сервисов.
- **Задачи**: цель, неделя, дедлайн, ответственный, статус, приоритет (Низкий / Средний / Высокий / Критический), метки, чек-листы, комментарии, вложения (файлы).
- **Обязательные поля** — при создании новой задачи нельзя сохранить без ответственного и срока.
- **Индикаторы на карточках** — просроченные задачи (дедлайн прошёл, задача не завершена) и «зависшие» задачи (статус давно не менялся, порог настраивается).
- **WIP-лимиты** — ограничение числа задач в колонке «В процессе» и др., настраивается в «Настройках».
- **Автоархивация** завершённых задач спустя заданное число дней.
- **Календарь** — все задачи с дедлайнами на месячной сетке.
- **Уведомления внутри приложения** — колокольчик с событиями (назначение, изменение статуса, комментарии и т.п.).
- **Еженедельная сводка** (только для администратора) — завершено за неделю по отделам, список просроченных задач, загрузка по сотрудникам с подсветкой перегруженных.
- **Аналитика и отчёты** — сводные показатели по отделам, экспорт отчётов в Excel.
- **Резервное копирование** — скачивание полной копии базы данных (SQLite) одной кнопкой.
- **Тёмная и светлая тема.**

Из выручкообразующих отделов в аналитике учитываются только «Продажи» и «Закупка товара» — остальные отделы (склад, касса, курьеры, гравировка) не влияют на расчёт выручки.

## Технологии

- **Backend:** Node.js (Express), TypeScript, SQLite (`better-sqlite3`) + Drizzle ORM
- **Frontend:** React, Vite, Tailwind CSS, shadcn/ui, TanStack Query, wouter (роутинг), dnd-kit (drag-and-drop)
- **Аутентификация:** логин/пароль, токен-сессия на сервере (без localStorage/cookies — обновление страницы разлогинивает, это осознанное решение)

## Требования

- Node.js 20+
- npm

## Тестирование в облачной среде (GitHub Codespaces)

В репозитории уже настроен `.devcontainer/devcontainer.json` — можно проверить проект без установки чего-либо на свой компьютер:

1. Откройте репозиторий на GitHub.
2. Кнопка **Code → Codespaces → Create codespace on main**.
3. Среда автоматически поднимется и выполнит `npm install`.
4. В терминале запустите `npm run dev` — GitHub предложит открыть приложение на порту 5000 в браузере.

Бесплатный лимит личного аккаунта — 120 core-часов в месяц (≈60 часов на 2-ядерной машине).

## Установка и запуск (разработка)

```bash
git clone https://github.com/disarmint/AppKanban.git
cd AppKanban
npm install
npm run dev
```

Приложение поднимется на `http://localhost:5000` (backend и frontend — на одном порту). При первом запуске без файла базы данных автоматически создаётся `data.db` и заполняется тестовыми данными (сиды).

### Учётные данные администратора при первом запуске

```
Логин: admin
Пароль: kanban2026
```

Рекомендуется сменить пароль после первого входа в разделе «Пользователи».

## Запуск в продакшене

```bash
npm run build      # собирает frontend (Vite) и backend (в dist/index.cjs)
npm start           # NODE_ENV=production node dist/index.cjs
```

По умолчанию сервер слушает порт из переменной окружения `PORT` (или 5000, если не задана).

## Проверка типов

```bash
npm run check   # tsc --noEmit
```

## Работа с базой данных

- Файл базы — `data.db` в корне проекта (в `.gitignore`, не хранится в репозитории).
- Миграции — в папке `migrations/`, применяются автоматически при старте сервера.
- Изменение схемы: `shared/schema.ts` → `npm run db:push` (Drizzle Kit).
- Резервная копия — через UI («Настройки → Резервное копирование») либо копированием файла `data.db` (использует `VACUUM INTO` для целостной копии на «горячей» базе).

## Структура проекта

```
client/       — React-приложение (frontend)
server/       — Express API, авторизация, cron-задачи, сиды
shared/       — общая схема данных (Drizzle) и утилиты дат
migrations/   — SQL-миграции базы данных
script/       — сборочные скрипты
uploads/      — загруженные файлы-вложения (не хранится в git)
```

## Заметки

- Приложение пока не размещено на постоянном публичном адресе — работает локально или на выбранном вами сервере/хостинге.
- Данные хранятся только в собственной базе SQLite — интеграции с Google Sheets нет.
- Все уведомления — только внутри приложения (без email/push).
